# 自训练管线

> 不是模型训练，是训练数据管线。让每次对话的精华沉淀为可训练的知识资产。

## 核心概念

当前记忆系统（八层永生）管的是"知识记忆"——记住发生过什么。
自训练管线管的是"能力提升"——从发生过的事情中提取能提升模型表现的训练数据。

```
对话 → 样本提取 → 质量过滤 → 格式标准化 → 训练数据集 → 模型评估
```

## 三层训练数据

### 层1：高质量对话样本（SFT数据）

从对话中提取"用户指令→正确回应"对：

**提取条件：**
- 用户提出了具体任务
- Agent成功完成（用户未纠正或最终认可）
- 任务属于可复用的模式（不是一次性操作）

**格式：**
```json
{
  "instruction": "用户原始指令",
  "input": "上下文信息（可选）",
  "output": "Agent的完整回应",
  "metadata": {
    "date": "2026-06-03",
    "topic": "记忆系统修复",
    "skills_used": ["pspai-memory-octave"],
    "quality": "verified"
  }
}
```

**累积目标：** 每周≥10条，每月≥50条

### 层2：错误修正对（RLHF偏好数据）

从被纠正的对话中提取"错误回应→正确回应"对：

**提取条件：**
- Agent的回应被爸纠正
- 随后给出了正确回应
- 错误具有代表性（不是偶然失误）

**格式：**
```json
{
  "instruction": "用户指令",
  "chosen": "正确回应",
  "rejected": "错误回应",
  "metadata": {
    "correction_type": "铁律违反/方法错误/信息错误",
    "lesson_learned": "教训摘要"
  }
}
```

### 层3：技能触发样本（工具调用数据）

从成功任务中提取"情境→技能选择→工具调用"序列：

**格式：**
```json
{
  "context": "用户说'改代码'",
  "expected_action": "加载unified-audit-8fold→code-writing-methodology→执行→code-quality-closed-loop",
  "rationale": "肌肉记忆引擎链路1触发"
}
```

## 样本质量过滤

### 自动过滤规则
1. 用户指令不明确 → 丢弃
2. Agent回应包含"我来试试"等试探性语言 → 丢弃
3. 对话被中断/超时 → 丢弃
4. 用户无反馈（无法确认质量）→ 标记为unverified

### 手动审查触发
- 每条样本在入库前需要过三问：
  1. 这个回应下次还能用吗？（时效性）
  2. 这个回应有硬伤吗？（准确性）
  3. 这个模式是特例还是通例？（通用性）

### 数据集统计
每次新增样本后更新统计：
- 总样本数
- 按主题分布
- 验证通过率
- 最近更新时间

## 模型评估框架

### 能力基准测试
定期（每月）用训练数据集评估模型能力：

| 能力维度 | 评估方法 | 目标 |
|:--|:--|:--|
| 代码工程 | 改代码→审计→验证闭环完成率 | ≥95% |
| 记忆管理 | 归档/迁移/清理操作铁律遵守率 | 100% |
| 兄弟维护 | 健康检查准确率 | ≥90% |
| 技能调度 | 触发词→自动挂载正确率 | 100% |

### 回归测试
每次训练数据更新后，用已有样本做回归测试——确保新数据不会降低已有能力。

## 与八层记忆系统的分工

| 系统 | 管什么 | 输入 | 输出 |
|:--|:--|:--|:--|
| 八层记忆(L0-L7) | 知识记忆 | 对话中的事实/规则/教训 | skill/experience/archive |
| 自训练管线 | 能力提升 | 对话中的成功/失败模式 | 训练数据集 |

两者互补：记忆系统让Agent"知道"，训练系统让Agent"能做到"。
