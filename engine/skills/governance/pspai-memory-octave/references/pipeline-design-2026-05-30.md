# PSPAI 记忆管线设计 — 2026-05-30 改制

> 刘玉龙P07 x 刘勇（爸）全天讨论产物

## 管线时序

```
每天凌晨:
  01:00  反刍（回顾当天记忆，归档打标签）        ~5分钟
  01:10  L4 知识·经验·教训提炼（三维，≥1触发）   ~8分钟
  01:18  L6 悟道觉醒（跨层模式发现）              ~10分钟

每周一:
  01:28  L7 推陈出新（用悟道标准清理冗余）        ~10分钟
```

## 设计原则

1. **材料→提炼→悟道→清理** — 逻辑链不可逆
2. **十分钟间隔** — 上一步结果充分沉淀后再启动下一步
3. **反刍后立即提炼** — 信息最新鲜，不隔夜
4. **悟道在提炼之后** — 需要三维材料(知识+经验+教训)才能发现深层模式
5. **去冗在悟道之后** — 用悟道校准后的标准判断冗余，不是静态规则
6. **放手学** — L4≥1即触发，不怕产出多，L6/L7负责兜底

## L4 三维提炼

### 预运行（每次提炼前，~2分钟）

**陷阱：MEMORY_INDEX.md ≠ filesystem 真实情况。** L1→L3 迁移可能创建了 archive 但忘记更新索引。

1. `search_files(target='files', pattern='*.md', path='archive/')` → 获取实际归档列表
2. 比对 MEMORY_INDEX.md → 将遗漏文件加入标签目录树 + 时间倒序表
3. 更新索引头部统计数（归档总数）

**验证：** 预运行后归档总数 = 实际文件数

### 提炼运行（~8分钟）

| 维度 | 回答 | 标注 |
|:---|:---|:---|
| 新知识 | 学到了什么新事实/概念/认知 | [知识] |
| 成功经验 | 什么做对了、可复用的模式 | [经验] |
| 失败教训 | 什么做错了、根因、如何避免 | [教训] |

三维缺一不可。知识="是什么"，经验="怎么做对"，教训="别怎么做"。

## L6 悟道觉醒

- **实时触发：** L4/L5/L7 调用 `self.l6.trigger(source, message, importance)`
- **觉醒等级：** 青铜(3)→白银(10)→黄金(25)→钻石(50)→星耀(100)→王者(200)
- **Cron补充：** 每天01:18深层扫描，发现实时触发遗漏的跨层模式

## L7 推陈出新

- **驱动源：** L6悟道产出新标准后驱动L7清理
- **安全机制：** 标记→retired/→7天观察→真删
- **不可恢复：** 即便删除也保留在冷存储中

## L6↔L7 功能调换

| | 旧 | 新 |
|:---|:---|:---|
| L6 | 推陈出新(去冗) | 悟道觉醒(发现模式) |
| L7 | 灵魂升华(觉醒) | 推陈出新(去冗) |
| 类名 | L6Renewal → L6Enlightenment | L7Sublimation → L7Renewal |

**逻辑：** 先悟道(需要完整材料)→后去冗(需要悟道校准的标准)。先去冗再悟道=用旧标准筛材料→悟道只能用二手数据。
