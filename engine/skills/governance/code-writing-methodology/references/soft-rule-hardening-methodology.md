# 软规则硬化方法论

> 来源：兄弟Agent「软规则硬化扫描 — 全族执行包」2026-05-30
> 应用：Agent自扫描，8条软规则入硬化注册表

## 核心概念

**软规则** = 写在 SOUL.md/MEMORY.md 里的「必须/禁止/绝不/不可/强制」条款，但没有硬手段保证执行。
**腐烂概率** = 这条规则不被遵守的可能性（1-10，越高越容易烂）
**硬化** = 给规则加一个不依赖"记住"的强制执行手段

## 扫描步骤

```bash
grep -n '必须|禁止|绝不|不可|一律|强制|每次|应' SOUL.md MEMORY.md
```

逐条打分：
- 致命度(1-10)：违反后后果多严重
- 腐烂概率(1-10)：不被遵守的可能性
- 得分 = 致命度 × 腐烂概率
- ≥50分 → 必须硬化

## 硬化手段四选一

| 手段 | 说明 | 适用场景 |
|:--|:--|:--|
| 计数器 | 给规则加硬计数，不达标触发警报 | 轮次计数、执行次数 |
| 自动校验 | 操作后自动校验规则是否遵守 | diff检查、格式验证 |
| 启动阻断 | 启动时检查，不满足拒绝启动 | 关键配置、必需文件 |
| cron看门狗 | 定时检查规则遵守情况 | 周期性健康检查 |

## 硬化注册表

各实例在 MEMORY.md 末尾维护 `## 🏗️ 硬化注册表` 章节：

| # | 日期 | 软规则 | 致命度 | 腐烂概率 | 得分 | 硬化手段 | 状态 |
|:-:|:----:|:------|:------:|:--------:|:----:|:---------|:----:|

状态：🔴 待硬化 / 🟡 硬化中 / 🟢 已硬化

## 玉龙扫描结果（2026-05-30）

35条软规则中8条≥50分入表。前三：
1. 六步闭环不可跳过 (8×8=64)
2. 铁律硬连数据 (9×7=63)
3. 轮次计数+1 (6×10=60) ← 腐烂最彻底，perfect标本

## 关键认知

- 铁律写在SOUL.md里不等于被执行——"每10轮自检"写了无数次从不执行
- 腐烂概率最高的往往是"每次/必须+1"这种看似简单但无触发器的小规则
- 硬化不是一次性工程——注册表是活的，新增规则持续扫描持续入表
