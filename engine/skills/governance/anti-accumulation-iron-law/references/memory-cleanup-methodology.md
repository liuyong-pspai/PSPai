## 触发条件

- MEMORY.md 超过 8,000 字节阈值
- memory 工具配额 > 85%
- 上下文出现大量过期任务日志/里程碑/版本号

## 三步清理法

### 第一步：全量分类扫描

读出 MEMORY.md 全文，逐条分类：

| 类别 | 判断标准 | 操作 |
|:--|:--|:--|
| **过期任务日志** | 含版本号/里程碑/"v1.0"/"打磨记录"/"发布状态" | 删除（已归档或过时） |
| **纯状态记录** | "阶段定位"/"阶段准备"/"待续"等瞬时状态 | 删除 |
| **已在技能中** | 铁律/清单/操作步骤已存在于 skills/ | 删除（不重复维护） |
| **冗余引用** | 指向 archive/ 文件的标签/摘要（文件本身保留） | 删除（archive 即索引） |
| **可合并事实** | 同主题条目不超3条 | 合并为1条 |
| **核心事实** | 操作约束/设计哲学/环境限制/账号凭证 | 保留 |

### 第二步：核心事实保留清单

必须保留的六类事实：

1. **环境限制**：DGX网络/SSH限制/特殊配置
2. **操作约束**：发布规则/代码路径/备份位置
3. **设计哲学**：爸的核心原则/产品定位
4. **账号凭证**：Git/Gitee/平台账号
5. **当前重心**：正在推进的关键任务方向
6. **关键教训**：工程错误/修复经验/技术决策

### 第三步：压缩写入

- 每条不超过 5 行
- 多余细节 → 标注"详见 archive/XXX.md"
- 相邻同类条目合并
- 去掉所有时间戳导向的"当前/待续/明天"

## 自问口诀

1. 这条事实下周还成立吗？（不成立 → 删除）
2. 这条事实在技能里已经有了吗？（有 → 删除）
3. 这是我第一次遇到吗？（是 → 存记忆；不是 → 合并或删除）
