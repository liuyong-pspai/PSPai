# P0 执行断层：L6→行动的三阶层问题分区

> 来源：L6悟道#003（2026-06-02）跨层全量审计
> 关联：pspai-memory-octave L6 节「⚠️ P0 执行断层」
> 状态：设计文档，待爸审批后实施

---

## 问题定义

L6 悟道引擎每次运行产出 N 个跨层发现和 P0/P1/P2 行动建议。但这些建议并不平等——它们天然分为三类，每一类需要不同的执行路径。**当前 cron 管线只能覆盖类型A，类型B和C在 cron 周期内悬空，形成「背诵循环」——L6 每轮发现同样的问题，但始终无人执行。**

---

## 三阶层分区模型

```
类型A：L7可修复（文件重构/索引更新/指针清理/内容搬迁/归档创建）
  能力边界：patch/write_file/read_file/search_files 可完成
  执行率：~100%（cron L7可直接执行）
  示例：
    - MEMORY.md 内容拆分→创建L3归档+替换短指针
    - 索引遗漏补录
    - 短指针去重合并
    - 孤儿归档/悬空索引修复

类型B：需代码改动（脚本修改/trigger管线/工具注册/config变更）
  能力边界：cron只能写markdown文件不能改.py并重启验证
  执行率：0%（cron无法完成）
  示例：
    - l1_memory_guard.py 增加 check_file 模式
    - L4→L5 trigger管线打通（cron路径通知L5）
    - 新工具注册 + 网关重启
    - config.yaml 参数变更

类型C：需爸批准（阈值变化/架构决策/新技能创建/铁律修订/策略变更）
  能力边界：需要人类判断和授权
  执行率：0%（等待下次对话）
  示例：
    - L5 自动触发阈值从≥3次降为≥24h+3000字
    - L7 增加每日渐进清理模式
    - 三段式工程法正式命名+创建skill
    - L1→L3 迁移条件扩展（文件大小双触发）
```

---

## 背诵循环

**定义**：同一 P0 项被 L6 连续 2+ 轮标记但始终未解决。

**已发生的背诵循环实例**：

| P0 项 | 首现 | 重复轮次 | 类型 | 状态 |
|:-----|:-----|:--------:|:----:|:----:|
| l1_memory_guard.py check_file | L6#002 (6/01) | L6#003 (6/02) | B | 悬空 |
| L4→L5 管道打通 | L6#002 (6/01) | L6#003 (6/02) | B | 悬空 |
| MEMORY.md 清单独立化 | L6#002 (6/01) | L6#003 (6/02) | A | L7可执行但未全部完成 |
| 三段式工程法正式命名 | L6#002 (6/01) | L6#003 (6/02) | C | 等爸批准 |

**危害**：P0 项从「需要修的 bug」退化为「已知的技术债务」→ 心理熵增 → L6 产出价值贬值。

---

## P0 升级通道设计

### 机制 1：MEMORY.md 告警注入

当 L6 检测到同一 P0 项连续 2 轮未解决时：
- 在 MEMORY.md 头部注入告警标记，下次对话启动时 LLM 自动看到
- 格式：
```markdown
## ⚠️ L6 P0 告警（{日期} 悟道#NNN 标记）: {问题简述}
→ 详见 archive/{悟道笔记文件} §发现N
连续 {N} 轮未解决。类型{B/C}，需{代码改动/爸批准}。
```
- 告警在问题解决后由 L6 自动清除

### 机制 2：L7 patches/ 目录

对于类型B问题，L7 可生成精确的 `patch` 操作指令：
- 写入 `L7/patches/YYYY-MM-DD-{描述}.patch` 
- 内容为精确的 `old_string` → `new_string` 对，供下次对话中直接 `skill_manage(patch)` 执行
- 限制：仅适用于已知文件、已知内容的确定性改动（如 l1_memory_guard.py 增加 check_file 方法）

### 机制 3：对话启动优先级队列

下次对话启动时：
1. 检查 MEMORY.md 中是否有 `.l6_p0_alert` 标记
2. 检查 `L7/patches/` 中是否有待执行的补丁
3. 按优先级排序：P0 告警 > P0 patches > P1 建议 > 常规对话

## 设计原则

1. **升级而非重复**：不要求 cron 具备代码修改能力——那不安全。而是在「发现」和「对话」之间建立升级通道
2. **最小侵入**：告警标记只在 MEMORY.md 中增加 1-3 行，不改变现有 cron 管线
3. **可回滚**：告警标记由 L6 管理（写入+清除），不在代码层硬编码
4. **分层授权**：类型A→L7自主、类型B→自动生成patch+对话执行、类型C→告警升级到爸
