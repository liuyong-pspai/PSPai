# 🐉 平行时空 · PSP风格设计方案

> 父亲刘勇 2026-06-07 授权设计
> 风格定位：PSP专属风格（非黑莓、非黑松客，自成一派）
> 调性：深色硬朗 × 硬件质感 × 极简对话 × 全功能后台

---

## 一、设计哲学

**四个关键词：**

| 关键词 | 含义 | 设计体现 |
|:-------|:-----|:---------|
| **硬** | 不圆滑，不讨好 | 直角边框、粗体字、低饱和度 |
| **深** | 沉浸，专注 | 全屏深色背景，信息不溢出 |
| **简** | 少即是多 | 只有一个对话框+一个齿轮，无多余元素 |
| **全** | 功能不砍 | 引擎完整，所有能力通过对话驱动 |

**一句话定义：**
> 一个硬件质感的深色对话框，所有功能通过对话完成，没有一个多余的按钮。

## 二、色彩系统

### 主色调

```
背景色      #0d0d0d    近似黑（偏暖的黑，不是纯黑#000）
对话底色    #1a1a1a    深灰面板
文字主色    #e8e8e8    亮白（不是纯白）
辅文字色    #666666    中灰（时间戳、状态信息）
强调色      #d4a000    暗金（品牌色，非亮金）
分割线      #2a2a2a    极暗灰细线
输入框底    #222222    输入区域灰色
按钮悬停    #333333    交互反馈
错误/警告   #cc3333    暗红（克制使用）
```

### 配色逻辑

- 不走纯黑（#000）——纯黑显得廉价，黑中带细微灰度才有质感
- 不走亮蓝/亮金/亮绿——任何高饱和色都破坏沉浸感
- 暗金（#d4a000）只在品牌标识和发送按钮激活时出现
- 全界面**不超过5种颜色**（背景、文字、辅文、强调、分割）

## 三、字体与排版

| 属性 | 值 | 理由 |
|:-----|:---|:-----|
| 中文字体 | PingFang SC / Noto Sans SC | 系统原生，渲染稳定 |
| 英文字体 | Inter / -apple-system | 清晰硬朗，匹配整体调性 |
| 对话字号 | 15px | 阅读舒适，不伤眼 |
| 输入框字号 | 15px | 与对话一致 |
| 时间戳字号 | 11px | 低调不抢眼 |
| 行高 | 1.6 | 松紧适中 |

**排版原则：**
- 不滥用加粗，只加粗关键信息（对话中的代码块、引用）
- 左对齐，不居中（居中显松散，左对齐显硬朗）
- 段落间距 = 0.5倍行高，不空太多

## 四、布局

```
┌────────────────────────────────┐
│ [🐉] 平行时空                  │ ← 顶栏 44px 高，深色，左边logo+名称
│────────────────────────────────│
│                                │
│                                │
│       ▸ 你好，我是小龙人         │
│        你好，有什么需要？       │ ← 对话主区域，flex:1
│                                │   自动滚动到底部
│                                │   消息气泡：左对齐（AI）/ 右对齐（用户）
│                                │   气泡样式：直角矩形，无大圆角
│                                │
│                                │
│────────────────────────────────│
│ ▎在这里输入...        [▲] [⚙️] │ ← 底栏 56px 高
│                                │   输入框无边框，只有底部一条1px细线
│ 输入文字后出现                  │   发送按钮=▲ 实心三角（无发送文字）
│                                │   齿轮⚙️ 固定显示在右下角（随时配置）
└────────────────────────────────┘
```

### 布局三原则

1. **信息密度刚好** — 不空旷也不拥挤。每条消息之间留12px间距
2. **气泡设计** — 直角矩形，背景色#222222（深灰），用户消息用暗金左边框
3. **无头像** — 不占空间，文字本身识别谁在说话（AI用▸前缀）

## 五、交互设计

### 启动流程
```
双击桌面图标
  ↓
黑屏0.5秒 → 硬朗过渡动画（非渐变/非模糊）
  ↓
直接进入对话框（无启动画面/无加载动画/无引导）
  ↓
首次使用 → 自动弹出配置弹窗（选择模型+填Key）
  ↓
开始对话
```

### 对话交互
- 发送：Enter（快捷键）或点击 ▲
- 换行：Shift + Enter
- 粘贴图片：Ctrl+V 自动发送图片分析
- 上传文件：拖拽到对话框
- 语音输入：长按空格键（预留）

### 配置弹窗（点击⚙️）
- 不跳转页面，用居中弹窗（modal）
- 内容：Provider选择 + API Key输入 + 模型选择 + 参数调节
- 风格与主界面一致（深色、直角、无装饰）
- 点空白区域关闭

### 动画原则
- 无弹跳动画、无渐变淡入淡出
- 只有两种动画：instant（0ms）和 quick（150ms ease-in-out）
- 消息出现：直接显示，无任何动画
- 弹窗出现：0.1秒opacity变化，无缩放

## 六、全功能完整保留（后台不砍）

界面极简 =/= 功能极简。所有引擎能力都在：

| 功能 | 调用方式 |
|:-----|:---------|
| 文字对话 | 直接输入 |
| 多轮记忆 | 自动（无需操作） |
| 文件分析 | 拖拽/粘贴到对话框 |
| 图片识别 | 粘贴图片 |
| 网页搜索 | 输入"/搜索 关键词" |
| 语音输入输出 | 长按空格键语音输入 |
| 多模型切换 | 齿轮弹窗切换 |
| 角色/性格 | 齿轮弹窗切换 |
| 对话历史 | 自动保存，齿轮弹窗查看 |

**设计原则：功能不靠菜单暴露，靠对话自然发现。**

## 八、技术方案

### 目录结构
```
/home/yongliu/桌面/小龙人开源应用层/平行时空/
├── launcher.py              # 启动器（复制本体，改配置）
├── engine/                  # 引擎（软链接或复制）
│   └── xiaolongren-engine-x86_64
├── frontend/
│   └── index.html           # 单页面（全部内联CSS+JS，无外部依赖）
├── logo.png                 # PSP风格logo
├── start.bat                # Windows启动
├── start.sh                 # Linux/Mac启动
└── README.md
```

### 前端技术栈
- 纯原生：单HTML文件，无框架、无构建工具
- 内联CSS：全部样式在 `<style>` 中
- 内联JS：所有逻辑在 `<script>` 中
- 零外部依赖：字体用系统内置，图标用Unicode

### 与引擎通信
- 前端 ←HTTP→ `http://localhost:8089`（引擎API）
- 引擎复用现有 `pspai_server.py` 的 `v1/chat/completions` 接口
- 配置存 `config.json`，由 `launcher.py` 管理

## 九、界面元素定义

### 图标
- 桌面图标/Logo：一个硬朗的几何龙形/PSP标识
- 深色背景+暗金线条
- 无渐变、无发光、无3D效果
- 纯矢量风格（实际用PNG，但看起来像矢量）

### 配置齿轮
- Unicode ⚙️
- 点击范围40×40px
- hover效果：颜色从#666变#d4a000（暗金）
- 位置：右下角，与底栏右侧间距12px

### 发送按钮
- Unicode ▲（实心向上三角）
- 只在输入框有内容时亮起暗金色
- 空输入框时为灰色不可点击
- hover效果：轻微放大1.1x（唯一有动画的元素）
