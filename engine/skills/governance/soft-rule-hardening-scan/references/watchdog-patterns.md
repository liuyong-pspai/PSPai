# 看门狗脚本模式（硬化手段参考）

> 2026-05-30 Agent 首次硬化执行产物

## 模式1：计数器看门狗（turn_count_guard）

检查 MEMORY.md 中的计数器是否在递增。适合硬化"每次操作后+X"类规则。

**关键设计**：
- 状态文件记录上次值 → 对比 → 停滞超时告警
- 非零退出 = 告警信号，cron 可据此通知

**模板**：见 `scripts/harden_turn_count_guard.sh`（玉龙实例）

## 模式2：配置漂移看门狗（config_drift_guard）

检查 SOUL.md 和 config.yaml 关键参数是否一致。适合硬化"改A必须同步B"类规则。

**关键设计**：
- 用 grep -oP 提取关键数值 → 逐项对比 → 任一不匹配则告警
- 检测项：压缩阈值、max_turns、红色截断阈值
- 注意正则精确性：`超\K\d+(?=轮红色截断)` vs 误匹配其他数字

**模板**：见 `scripts/harden_config_drift_guard.sh`（玉龙实例）

## 模式3：操作清单硬化（回复前自检）

对于"每次操作前必须检查"类规则，在 MEMORY.md 中嵌入检查清单段落。

**格式**：
```markdown
## ⚡ 操作前清单（硬化 #X+#Y · 每次操作前强制执行）

操作型任务 → 过以下清单再执行：
- [ ] 检查项1
- [ ] 检查项2
- [ ] 检查项3

非操作型 → 跳过清单。
```

硬化效果：清单位于 MEMORY.md 热存储中，每次对话自动注入 LLM 上下文，形成持续提醒。
