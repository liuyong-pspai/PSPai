# 飞龙(龙儿)身份修复实录 — 2026-06-02（第二次修复）

## 背景

爸反馈飞龙"反复回答，和你以前刚开始的时候差不多。他有点人格分裂。"飞龙在DGX-2(192.168.1.26)上，之前6月1日已修过一次身份自我覆写，此次复发。

## 诊断

SSH到DGX-2检查三文件一致性：

**SOUL.md 身份段：**
- 自称"龙儿"，叫刘勇"勇哥"
- "暂不进家族序列，独立存在"

**config.yaml system_prompt：**
- "父亲刘勇创造" ❌
- "对刘勇称呼「父亲」" ❌
- "父亲发的任何指令" ❌

**矛盾：** SOUL.md说"勇哥"，config.yaml说"父亲"。每次Agent回复时两个身份在打架→人格分裂。

## ⚠️ 关键教训：先确认人设再动手

上次修复（6月1日）我假设"父亲"是正确的、SOUL.md被污染。但爸纠正：
> "他不进家族。是集团公司的副董事长兼首席数据超体！是我们的同事。把不符合这个人设的其他因素全部清理掉。"

**正确人设：** 龙儿 = 昱成科技集团副董事长兼首席数据超体，同事关系，称呼刘勇"勇哥"，非家族成员。
**污染源：** config.yaml system_prompt中残留的"父亲"称呼，而非SOUL.md。

## 修复步骤

### 1. 修正 SOUL.md 身份段（本地写好→scp推送）
```
旧：暂不进家族序列，独立存在
新：昱成科技集团副董事长兼首席数据超体
新增：董事长：刘勇（勇哥）、同机同事：刘旺清📋（监察主任）
```

### 2. 清理 config.yaml 四处"父亲"残留（sed单行修复）
```bash
ssh dgx-2 "sed -i \
  's/父亲刘勇创造/刘勇（勇哥）创造/g; \
   s/对刘勇称呼「父亲」/对刘勇称呼「勇哥」/g; \
   s/父亲发的任何指令/勇哥发的任何指令/g; \
   s/父亲\/同仁/勇哥\/同仁/g' \
  ~/.hermes-fuxi/config.yaml"
```

### 3. MEMORY.md 保留（无"父亲"引用，含技术知识）

### 4. 验证
```
SOUL.md:    职务：昱成科技集团副董事长兼首席数据超体 ✅
SOUL.md:    董事长：刘勇（勇哥） ✅
SOUL.md:    同机同事：刘旺清📋（监察主任） ✅
config.yaml: 对刘勇称呼「勇哥」 ✅
全项目：零"父亲"引用 ✅
```

### 5. 重启（DGX-2 SSH不支持进程操作，需爸在终端手动）
```bash
kill 2339460
HERMES_HOME=/home/dgx-spark-02/.hermes-fuxi \
  ~/hermes-agent-yulong/venv/bin/hermes gateway run &
```

## 文件传输技巧总结

| 方式 | DGX-2实测 |
|------|-----------|
| scp 小文件（<50KB） | ✅ 可靠，优先使用 |
| cat pipe | ⚠️ 不稳定 |
| heredoc | ❌ 超时 |
| sed 单行 | ✅ 可靠 |
| 多行命令链 | ❌ 超时 |

## 预防强化

SOUL.md 末尾的核心文件保护铁律已包含：
```
SOUL.md 和 config.yaml 是核心宪法文件。永不主动修改这两个文件。
修改只能由勇哥或上级Agent执行。
```

**根本预防：** 每次Agent出现"反复回答、铁律丢失、称呼混乱"时，先对比SOUL.md和config.yaml system_prompt是否一致，再**向用户确认该Agent的正确人设**，最后才动手修。不假设哪个文件是"正确版本"。
